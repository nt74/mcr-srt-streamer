SRT Parameter Reference (Based on Haivision SRT Deployment Guide v1.5.x)

Round Trip Time (RTT) is the time it takes for a packet to travel from a source to a destination and back again[cite: 313]. It provides an indication of the network distance between endpoints[cite: 313].

  • Can be measured using the `ping` command or observed in SRT statistics[cite: 317, 319].
  • Typical RTTs (examples from guide): LAN (< 1ms), Continental US (60-100ms), Transoceanic (60-200+ms)[cite: 314, 315].
  • RTT is used as a guide when configuring SRT Latency and Bandwidth Overhead[cite: 316].

SRT Latency determines the size of the buffer (in milliseconds) used for managing packet jitter and retransmissions[cite: 291, 377]. It's a fixed value (typically 80-8000 ms) set on the sender and receiver; the higher of the two values is used for the stream[cite: 377, 386].

It should be set based on RTT and network stability. The calculation involves an RTT Multiplier[cite: 320, 381]:

SRT Latency = RTT Multiplier * RTT [cite: 324, 383]

  • The RTT Multiplier reflects how many attempts SRT might make to resend a packet before dropping it, compensating for network congestion[cite: 321, 323, 325].
  • A higher multiplier (and thus higher latency) is needed for links with higher constant packet loss to increase the chance of recovery[cite: 328, 334].
  • A common rule of thumb for good networks (low loss) is a multiplier of ~4[cite: 325, 382].

Measures network congestion as the percentage of packets lost compared to packets sent[cite: 352]. It drives the Latency and Overhead calculations[cite: 299].

  • Constant Loss: Packets lost at a steady rate. Requires relatively low overhead but potentially higher latency for recovery[cite: 352, 328, 334].
  • Burst Loss: Multiple consecutive packets lost. Requires higher Bandwidth Overhead to recover the block of data quickly within the Latency window[cite: 326, 330, 340, 341]. Burst loss lasting longer than the SRT Latency setting will cause stream artifacts[cite: 359].
  • This tool's recommendations are primarily based on a constant loss model[cite: 350, 355].

Specifies extra bandwidth allocated above the stream's main A/V bitrate, used for SRT control packets and retransmitting lost media packets[cite: 362].

  • Calculated as a percentage of the A/V bitrate[cite: 364].
  • Noisier networks with higher packet loss require a higher overhead percentage[cite: 366].
  • The total required bandwidth (Stream Bitrate + Overhead) must be less than the available channel capacity[cite: 306, 373]. The guide recommends headroom (e.g., 0.75 * Channel Capacity > SRT Stream Bandwidth * (1 + Overhead/100))[cite: 307].
  • Burst loss requires significantly more overhead than constant loss, especially at low latencies[cite: 343, 348].

This table provides estimated starting points based on measured constant packet loss[cite: 350]. Burst loss may require higher overhead, especially at lower RTT multipliers[cite: 355].
